### Golang的net包中的套接字编程

套接字的意思是：

- 在操作系统网络堆栈中,用于标识一个“端点”的抽象资源。一个端点包括IP地址和端口号。
- 套接字通过端点来标识正在网络上进行通信的两个应用进程。

在这里，我们需要查看套接字是如何被创建和使用的。使用套接字编程，需要用户自己处理以下几个点：

- 连接管理（如 连接数过多）
- 数据封包和拆包（序列化和反序列化）
- 错误处理 （返回error或返回错误码)

#### TCP套接字编程

##### 服务端程序：

​	以下是一个简化的服务端程序示例，有以下几个问题，但还是可以姑且一试：

1. 当接收到相应请求后，服务端调用goroutine来处理，实现了异步处理，但程序未对使用的goroutine数量进行限制。
2. 未进行数据装包拆包，直接发送字符串、打印字符串。
3. 错误处理仅做了打印日志，未进行重试。

```golang
package main

import (
	"fmt"
	"log"
	"net"
)

func main() {
	fmt.Println("hello new")
	addr, err := net.ResolveTCPAddr("tcp", "127.0.0.1:8000")
	if err != nil {
		log.Panic(err)
	}

	listen, err := net.ListenTCP("tcp", addr)
	if err != nil {
		log.Panic(err)
	}

	for {
		conn, err := listen.AcceptTCP()
		if err != nil {
			continue
		}

		go handleConnectionForServer(conn)
	}
}

func handleConnectionForServer(conn net.Conn) {
	for flag := false; ; {
		buf := make([]byte, 1024)
		if !flag {
			if _, err := conn.Write([]byte(Usage())); err != nil {
				log.Println("Error: ", err)
			}
			flag = true
			continue
		}
		if length, err := conn.Read(buf); err != nil {
			writeResponse(parseRequest(""), conn)
		} else {
			req := string(buf[:length])
			if req == "exit" {
				break
			}
			writeResponse(parseRequest(req), conn)
		}
	}
}

func Usage() string {
	return "return usage."
}

func writeResponse(resp string, conn net.Conn) {
	if _, err := conn.Write([]byte(resp)); err != nil {
		log.Print("Error: ", err)
	}
}

func parseRequest(req string) (resp string) {
	fmt.Println(req)
	return
}

```



##### 客户端程序：

​	以下是一个简化的客户端程序，从命令行界面读入数据，然后向服务端发送请求。若服务端未启动，tcp双向连接无法建立， 客户端会报错并停止运行。

```golang
package main

import (
	"bufio"
	"fmt"
	"net"
	"os"
	"os/signal"
	"strings"
	"syscall"
)

var sig = make(chan os.Signal)

func main() {
	fmt.Println("hello client")
	RemoteAddr, err := net.ResolveTCPAddr("tcp", "127.0.0.1:8000")
	if err != nil {
		panic(err)
	}

	LocalAddr, err := net.ResolveTCPAddr("tcp", "127.0.0.1:8001")
	if err != nil {
		panic(err)
	}

	conn, err := net.DialTCP("tcp", LocalAddr, RemoteAddr)
	if err != nil {
		panic(err)
	}
	handleConnectionForClient(conn)
}

func handleConnectionForClient(conn net.Conn) {
	go signalMonitor(conn)
	Stdin := bufio.NewReader(os.Stdin)
	for {
		getResponse(conn)
		fmt.Print("[ random_w ]#")
		input, err := Stdin.ReadString('\n')
		if err != nil {
			fmt.Println(err)
		}
		input = strings.TrimSpace(input)
		if len(input) == 0 {
			continue
		}
		switch input {
		case "quit", "exit":
			sig <- syscall.SIGQUIT
		default:
			sendMsgToServer(conn, input)
		}
	}
}

func sendMsgToServer(conn net.Conn, msg string) {
	for {
		_, err := conn.Write([]byte(msg))
		if err == nil {
			break
		}
	}
}

func getResponse(conn net.Conn) {
	respByte := make([]byte, 1024)
	length, err := conn.Read(respByte)
	if err != nil {
		fmt.Println("[ server ]# 接收消息失败")
	}
	for line, str := range strings.Split(string(respByte[:length]), "\n") {
		if len(str) != 0 {
			if line == 1 {
				fmt.Print(fmt.Sprintf("[ server ]# \n%s\n", str))
				continue
			}
			fmt.Println(str)
		}
	}
}

func signalMonitor(conn net.Conn) {
	signal.Notify(sig, syscall.SIGQUIT, syscall.SIGKILL, syscall.SIGINT)
	select {
	case <-sig:
		_, _ = conn.Write([]byte("exit"))
		fmt.Println("\nGood Bye !!!!!")
		os.Exit(0)
	}
}

```

#### Unix套接字编程

UNIX域套接字的特点:

- 使用文件系统路径作为地址,适用于同一主机内的IPC
- 与TCP socket等网络通信不同,不依赖IP地址和端口
- 进程间能通过类似管道的方式直接通信,性能好于网络通信

##### 服务端程序

​	仅地址解析与之前不同。

```golang
	addr := &net.UnixAddr{Name: "mysocket", Net: "unix"}
	listen, err := net.ListenUnix("unix", addr)
```

#### 多播通信

注意，多播地址应在224.0.0.0 - 239.255.255.255中。其中:

- 224.0.0.0 - 224.0.0.255: 本地网络多播
- 224.0.1.0 - 238.255.255.255: 公众组多播
- 239.0.0.0 - 239.255.255.255: 站点本地多播

设置多播地址时应遵循地址使用规则。
