### 策略模式

![img](/img/20240103-策略模式学习笔记/01.jpg)

策略模式有一个环境类，提供环境信息，装配不同的策略。

使用策略模式可以干掉繁琐的if和switch，把判断内嵌在环境类中。当共同方法越多，策略模式的优点越明显，可以做到一次选择，直接使用。

【在同一个环境类中选择多个策略】策略模式可以有一个优化。它的环境类会根据上下文选择不同的具体策略，我们有时候可能需要选择两类策略，两类策略的抽象接口都不一样，通过环境类解析装配到环境类里，接着一个个调用。

【策略跟模板方法模式混合】策略模式也可以跟模板方法模式混合。策略模式需要定义一个抽象接口，具体的策略类要实现这个接口。但我们可以定义出抽象接口以后，继承抽象接口写一个抽象类，在方法里规定它需要调用的小方法，再把小方法放到具体实现类里实现。

#### 策略模式跟简单工厂的区别

简单工厂的实现类是一次性的，每次生成新的实例。但策略模式的具体实现类可以不是一次性的，每次都装配那几个固定实例。

#### 策略模式跟模板方法模式的区别

策略模式的使用需要有一个环境类来做帮手，通过环境类屏蔽所有具体实现。而模板方法模式则是直接使用具体类，只是程序员在写代码时需要遵循一些规则罢了。

#### 策略模式跟享元模式的区别

策略模式的要点在于环境类，对于不同环境类会装配不同的策略。而享元模式重点在于类的拿取，并不关注现在的环境。

#### 【例子】策略模式的简单举例

下面是一个策略模式的简单举例，通过StrategyEnv持有环境信息，装配不同的InterestStrategy。拓展时，需要改写StrategyEnv类和新增InterestStrategy类。

```java
package src.test.com.channon.designPattern;

import org.junit.jupiter.api.Test;

public class InterestCalculationByStrategyPattern {

    class StrategyEnv{
        private InterestStrategy strategy;
        private int originAmount;
        private int duration;
        public StrategyEnv(int originAmount, int duration){
            this.originAmount = originAmount;
            this.duration = duration;
            if(originAmount > 10000){
                strategy = BigInterestStrategy.getInstance();
            }else{
                strategy = SmallInterestStrategy.getInstance();
            }
        }

        public double calculateInterest(){
            return strategy.getInterest(originAmount, duration);
        }
    }

    interface InterestStrategy{
        public double getInterest(int originAmount, int duration);
    }

    public static class BigInterestStrategy implements InterestStrategy {
        private static BigInterestStrategy instance;

        private BigInterestStrategy() {
            // 私有构造函数
        }

        public static synchronized BigInterestStrategy getInstance() {
            if (instance == null) {
                instance = new BigInterestStrategy();
            }
            return instance;
        }

        @Override
        public double getInterest(int originAmount, int duration) {
            return originAmount * Math.pow(1.03, duration);
        }
    }

    public static class SmallInterestStrategy implements InterestStrategy {
        private static SmallInterestStrategy instance;

        private SmallInterestStrategy() {
            // 私有构造函数
        }

        public static synchronized SmallInterestStrategy getInstance() {
            if (instance == null) {
                instance = new SmallInterestStrategy();
            }
            return instance;
        }

        @Override
        public double getInterest(int originAmount, int duration) {
            return originAmount * Math.pow(1.01, duration);
        }
    }

    @Test
    public void test(){
        int originAmount = 5000;
        int duration = 10;
        StrategyEnv env = new StrategyEnv(originAmount, duration);
        System.out.println(env.calculateInterest());
    }
}
```



#### 【例子】复杂的策略模式

策略模式混合模板方法模式。TaskExecutorChooser获取环境数据并装配相应的fetcher、parser、sender。而fetcher、parser、sender因为是通用的方法，我希望它们执行处理、存进数据库、调起下一个任务。所以它们可以有一个共同的抽象父类AbstractTask，在AbstractTask里写好流程，把每一小步的具体实现留给子类。

这段代码还可以进一步优化，将创建出来的具体Fetcher、Parser、Sender交给Spring管理，或者使用享元模式管理，避免重复创建类造成的开销。但这里就偷个懒，不再作修改了。

```java
package src.test.com.channon.designPattern.strategyPattern;

import com.mysql.jdbc.exceptions.MySQLDataException;
import org.junit.jupiter.api.Test;

import java.time.Instant;
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

public class FileParserByStrategyPattern {

    @Test
    public void test(){
        // 可用 cronjob 唤起
        TaskExecutorChooser chooser = new TaskExecutorChooser("[Parser File]");
        chooser.doExecution();
    }
}

class TaskExecutorChooser{
    TaskExecutor fetcher;
    TaskExecutor parser;
    TaskExecutor sender;
    String data;

    public TaskExecutorChooser(String data){
        this.data = data;

        // 装配executor
        fetcher = new FetcherOne();
        parser = new ParserOne();
        sender = new SenderOne();
    }

    public void doExecution(){
        System.out.println(data + " started at " + Instant.now());
        String fetcherData = fetcher.doTask(data);
        String parserData = parser.doTask(fetcherData);
        String senderData = sender.doTask(parserData);
        System.out.println(data + " finished at " + Instant.now());
    }
}

interface TaskExecutor{
    public String doTask(String data);
    public void duplicateException(Exception e);
    public void createNextTask(String data);
}

abstract class AbstractTask implements TaskExecutor{

    public String doTask(String data){
        // preProcess
        List<Object> afterPre = preProcess(data);
        // doProcess
        List<Object> midProcess = doProcess(afterPre);

        // save into database
        if(midProcess == null || midProcess.size() == 0){
            return data;
        }
        for(Object o: midProcess){
            try{
                saveData(o);
            } catch (MySQLDataException e){
                duplicateException(e);
            }catch (Exception e){
                e.printStackTrace();
                throw e;
            }
        }

        // midProcess
        afterProcess(midProcess);

        createNextTask(data);

        return data;
    }

    abstract public List<Object> preProcess(String data);
    abstract public List<Object> doProcess(List<Object> objects);
    abstract public void saveData (Object o) throws MySQLDataException;
    abstract public List<Object> afterProcess(List<Object> objects);

    public void duplicateException(Exception e){
        e.printStackTrace();
    }

    public void createNextTask(String data){
        try{
            // 操纵数据库
            System.out.println("Save "+data+"'s next task in database.");
        }catch (Exception e){
            e.printStackTrace();
            throw e;
        }
    }
}

class FetcherOne extends AbstractTask {
    @Override
    public List<Object> preProcess(String data) {
        System.out.println("FetcherOne do preProcess");
        ArrayList<String> objects = new ArrayList();
        objects.add(data);
        return Collections.singletonList(objects);
    }

    @Override
    public List<Object> doProcess(List<Object> objects) {
        System.out.println("FetcherOne do doProcess");
        return objects;
    }

    @Override
    public void saveData(Object o) throws MySQLDataException {
        System.out.println("FetcherOne do saveData");
    }

    @Override
    public List<Object> afterProcess(List<Object> objects) {
        System.out.println("FetcherOne do afterProcess");
        return objects;
    }
}
class ParserOne extends AbstractTask {
    @Override
    public List<Object> preProcess(String data) {
        System.out.println("ParserOne do preProcess");
        ArrayList<String> objects = new ArrayList();
        objects.add(data);
        return Collections.singletonList(objects);
    }

    @Override
    public List<Object> doProcess(List<Object> objects) {
        System.out.println("ParserOne do doProcess");
        return objects;
    }

    @Override
    public void saveData(Object o) throws MySQLDataException {
        System.out.println("ParserOne do saveData");
    }

    @Override
    public List<Object> afterProcess(List<Object> objects) {
        System.out.println("ParserOne do afterProcess");
        return objects;
    }
}
class SenderOne extends AbstractTask {
    @Override
    public List<Object> preProcess(String data) {
        System.out.println("SenderOne do preProcess");
        ArrayList<String> objects = new ArrayList();
        objects.add(data);
        return Collections.singletonList(objects);
    }

    @Override
    public List<Object> doProcess(List<Object> objects) {
        System.out.println("SenderOne do doProcess");
        return objects;
    }

    @Override
    public void saveData(Object o) throws MySQLDataException {
        System.out.println("SenderOne do saveData");
    }

    @Override
    public List<Object> afterProcess(List<Object> objects) {
        System.out.println("SenderOne do afterProcess");
        return objects;
    }
}
```

#### 【例子】装配不定量策略类的策略模式

​	有一些情况需要根据环境信息装配不定量的具体类。每个处理器（具体策略类）都有相同的输入和输出类型，并且它们按顺序链接在一起进行处理。这样，每个处理器都有机会处理数据，并将处理后的结果传递给下一个处理器。这有点参考责任链模式的思想，但并不是责任链模式。

```java
package src.test.com.channon.designPattern.strategyPattern;

import org.junit.jupiter.api.Test;

import java.util.ArrayList;
import java.util.List;

public class DataWashByStrategyPattern {

    @Test
    public void test(){
        DataWasher washer = new DataWasher("HelloWorld");
        washer.doProcessor();
    }
}

class DataWasher{
    List<Processor> processors;
    String env;
    public DataWasher(String env){
        this.env = env;
        processors = new ArrayList<>();
        // 可根据环境信息配置，不用写死
        processors.add(new LowerCaseProcessor());
        processors.add(new Appender('c'));
        processors.add(new Appender('a'));
        processors.add(new Replacer('e', ' '));
    }

    public void doProcessor(){
        for(Processor p:processors){
            env = p.process(env);
        }
        System.out.println(env);
    }
}

interface Processor{
    public boolean isMatch(String data);
    public String process(String data);
}

class LowerCaseProcessor implements Processor{

    @Override
    public boolean isMatch(String data) {
        return data.length() > 0;
    }

    @Override
    public String process(String data) {
        if(isMatch(data)){
            return data.toLowerCase();
        }
        return data;
    }
}

class Replacer implements Processor{

    char origin;
    char post;

    public Replacer(char origin, char post){
        this.origin = origin;
        this.post = post;
    }

    @Override
    public boolean isMatch(String data) {
        return data.contains(Character.toString(origin));
    }

    @Override
    public String process(String data) {
        if(isMatch(data)){
            return data.replace(origin, post);
        }
        return data;
    }
}

class Appender implements Processor{

    char ch;

    public Appender(char ch){
        this.ch = ch;
    }

    @Override
    public boolean isMatch(String data) {
        return true;
    }

    @Override
    public String process(String data) {
        if(isMatch(data)){
            StringBuffer sb = new StringBuffer(data);
            sb.append(ch);
            return sb.toString();
        }
        return data;
    }
}

```
