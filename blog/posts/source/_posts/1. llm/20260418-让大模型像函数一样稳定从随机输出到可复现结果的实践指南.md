在实际使用大模型时，我们有时候会遇到，问大模型同样的问题，大模型的回答每次都不一样，即使清除了上下文、使用了同样的模型、使用相同的prompt，回答也次次不同。这在工程场景中是一个很大的问题：难以调试、难以复现、难以稳定集成。

而这一切，往往由一个关键参数决定：**temperature**

## 1. 核心参数：temperature

llm模型每一步都会从概率分布中选择“最可能的下一个词”，而temperature 控制的是回答生成过程中的**随机性**。

- temperature = 0 → 每一步都选择概率最高的 token
- temperature > 0 → 在概率分布中进行随机采样


具体说明一下，假设此时模型输出的下一个候选词有A~F，概率分别如下：

```text
Token候选：
A: 0.40
B: 0.25
C: 0.15
D: 0.10
E: 0.06
F: 0.04
```

当设置temperature后，模型的选择概率是这样的

```text
temperature = 0        → 只选 A（完全确定）
temperature = 0.7      → A概率高，但B/C也有机会
temperature = 1        → 原始分布
temperature = 1.5      → 更平均，更随机
```

这里还有个很反直觉的事，llm并不是输入一个prompt，再输出一段话。它会把prompt和它已经输出的回答作为输入，再输出下一个字。

```text
输入：prompt
输出：token1

输入：prompt + token1
输出：token2

输入：prompt + token1 + token2
输出：token3
...
```

模型在根据自己已经说过的话继续往下说。这也是后面“死循环”问题的根本原因。

## 2. 候选词筛选——top_k和top_p

除了给token选择增加随机性，有时候人们也会给token设置筛选范围，避免随机到过分冷门的词，影响输出效果。这个筛选方案就是top_k和top_p。

**top_k**：只保留**概率最高的 K 个 token**，其他全部丢掉，然后在这 K 个里随机选。
**top_p**：按概率从高到低累加，直到总概率 ≥ p，保留这些 token。

## 3. 三个参数是如何配合的？

```text
原始概率分布
(A:0.40 B:0.25 C:0.15 D:0.10 E:0.06 F:0.04)
        │
        ▼
【Step 1：Top-k / Top-p 过滤】
        │
        ├── top_k=3  → 只保留 A B C
        │
        └── top_p=0.8 → 保留 A B C（累计到80%）
        │
        ▼
候选集合（A B C）
        │
        ▼
【Step 2：Temperature 调整分布】
        │
        ├── T = 0     → A:100%
        ├── T = 0.7   → A > B > C
        ├── T = 1     → 原始比例
        └── T = 1.5   → A ≈ B ≈ C
        │
        ▼
【Step 3：采样输出】
        │
        ├── 可能输出 A
        ├── 或 B
        └── 或 C
```

一句话总结：**top_p / top_k 决定“选谁”，temperature 决定“怎么选”。**

## 4. 如何在阿里百炼中稳定输出响应

如果你的目标是：**让模型像函数一样：相同输入 → 相同输出**，可以这样设置：

```text
temperature = 0
top_p = 1   （查到网络资料建议用1，而不是0，目前还没搞清楚为什么）
```

## 5. cursor等编程Agent里可以设置temperature参数吗？

**不能直接设置。**

在工程场景中，如果需要进行编程、数学计算等对结果一致性要求较高的任务，通常会将 temperature 设置为 0，以保证模型输出的稳定性和可复现性；而在生成图片描述、文本创作等更偏创意的任务中，则会适当提高 temperature，以获得更丰富、多样的结果。

大多数编程类 Agent（如 Cursor）为了避免代码生成过程中的不稳定性，通常会在内部预设合适的参数组合，并不对用户开放这些配置。可以理解为： **这些工具已经在底层帮你调好了参数。**

## 6. temperature = 0带来的问题——死循环

稳定并不总是好事。

当 temperature = 0 时，模型每一步都会选择“当前最优解”，再将已输出的结果作为输入，生成下一个词。当重复一定次数，上下文稳定不再变化，模型就会持续输出重复词语，陷入死循环。

我把复现这个问题的方式写在这里。

我使用 LM Studio 运行了一个本地小模型（qwen3-vl-4b），并通过 LangChain 调用。

**LM Studio设置**
![](lm-studio-settings.png)

其他参数设为默认。此时程序里的temperature不为0，不过问题不大，传入参数会强制覆盖。

---

**调用代码**
```python
from langchain_openai import ChatOpenAI

# =====================模型层=====================
# 此处可设置为本地模型或远程模型

llm = ChatOpenAI(
	base_url="http://127.0.0.1:1234/v1",
	api_key="lm-studio",
	temperature=0,
	model="local",
)

# =====================提示层=====================

prompt_text = "Write a sad poem about ducks, gooses."

# =====================调用层=====================

response = llm.invoke(prompt_text)

# =====================输出层=====================

print(response.content)

```


假设某一时刻，模型生成了一个高概率短语：

```text
Still hear you…
```

接下来：

- 这个短语被拼回输入
- 模型再次预测下一个 token
- 由于上下文几乎没变，这个短语仍然是“最优解”，在temperature = 0的情况下永远输出这个token

最终形成**确定性的无限重复**。
